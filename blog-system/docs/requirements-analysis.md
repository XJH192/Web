# 需求分析文档 - 个人博客 / 多用户博客系统

## 1. 项目背景

个人博客系统用于用户发布文章、沉淀知识、进行评论互动，并由管理员进行内容维护。课程验收要求系统具备完整业务闭环、前后端交互、数据库设计和 AI 结合功能，因此本项目在 Hexo 主题基础上扩展动态博客功能，并使用 Spring Boot 提供后端接口。

## 2. 用户角色

- 游客：浏览文章、搜索文章、查看分类标签和评论。
- 普通用户：注册登录、发布文章、编辑/删除本人文章、发表评论和回复。
- 管理员：管理所有文章、审核评论、维护分类和标签、查看后台统计。

## 3. 功能需求

- 用户注册与登录：提供普通用户注册、管理员/用户登录、基于 token 的简单身份识别。
- 文章管理：文章发布、编辑、删除、列表查询、详情查看。
- 分类标签：分类和标签用于组织文章，管理员可维护。
- 评论回复：登录用户可评论和回复，AI 审核后自动标记状态。
- 搜索统计：支持按关键词、分类、标签搜索，文章详情访问时增加阅读量。
- 后台管理：管理员查看统计、文章、评论、分类、标签。
- AI 功能：大纲生成、摘要生成、标签推荐、评论审核、博客问答助手。

## 4. 非功能需求

- 本地可运行，前端和后端地址清晰。
- RESTful API，返回格式统一。
- 后端分层明确，Controller 不写主要业务逻辑。
- 表单具备基本校验，异常返回友好提示。
- 数据库表设计规范，后续可持久化。

## 5. 业务流程

```text
用户注册/登录
  -> 浏览或搜索文章
  -> 查看文章详情，阅读量 +1
  -> 发布或编辑文章
  -> 发表评论
  -> AI 评论审核
  -> 管理员进入后台
  -> 审核评论/管理内容/维护分类标签
```

## 6. 数据流程说明

前端通过 `blog-api.js` 发送 HTTP 请求到 Spring Boot。后端 Controller 接收 DTO，调用 Service 进行校验和业务处理，当前由 `InMemoryBlogRepository` 存储数据。后续接入 MySQL 时替换 Repository 层即可。

## 7. 数据库初步设计

详见 `database/schema.sql`，包括 users、categories、tags、articles、article_tags、comments、ai_usage_logs 七张表。

## 8. AI 功能需求

- 输入文章标题，输出文章大纲。
- 输入文章正文，输出摘要。
- 输入标题和正文，输出推荐标签。
- 输入评论内容，输出审核结果。
- 输入博客相关问题，输出基于现有文章内容的问答结果。

## 9. 小组成员分工建议

- 前端同学：Hexo 动态页面、表单校验、后台交互。
- 文档/AI 同学：数据库脚本、AI 模块、README、PPT、演示视频。
