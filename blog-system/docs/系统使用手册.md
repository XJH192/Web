# Ciallo～(∠・ω< )⌒☆ 多用户智能博客系统使用手册

> 文档版本：V1.0  
> 编制日期：2026 年 7 月 9 日  
> 适用对象：系统普通用户、管理员、部署人员和课程验收人员

## 1. 系统简介

Ciallo～(∠・ω< )⌒☆ 多用户智能博客系统是一个基于 Hexo、Spring Boot、MySQL 和 DeepSeek 的课程实训项目。

系统支持：

- 用户注册、登录和角色权限。
- 文章草稿、发布、编辑、删除、附件、目录、分类、标签和归档。
- 点赞、评论、回复和评论点赞。
- 用户搜索、文章搜索、资料卡、关注、粉丝和互关。
- 私信和互动通知。
- 每个账号独立的 8 张个人相册。
- AI 大纲、摘要、标签、分类和博客问答。
- AI 内容初审和管理员人工复核。
- 管理员用户、文章、评论、分类、标签和 AI 日志管理。

## 2. 运行环境说明

### 2.1 推荐环境

| 软件 | 推荐版本 |
|---|---|
| 操作系统 | Windows 10/11 |
| 浏览器 | Chrome 或 Edge 最新版 |
| Node.js | 18 或更高 |
| Java | JDK 17；源码编译目标兼容 Java 8 |
| Maven | 3.8 或更高 |
| MySQL | MySQL 8 |
| 数据库工具 | Navicat、MySQL Workbench 或命令行 |

### 2.2 项目端口

```text
前端端口：4000
后端端口：8080
数据库端口：3306
```

### 2.3 数据库初始化

1. 启动 MySQL 服务。
2. 打开 Navicat 或 MySQL 客户端。
3. 执行：

```text
blog-system/database/schema.sql
```

也可以执行：

```text
blog-system/database/mydataset_navicat.sql
```

脚本会创建 `mydataset` 数据库、15 张业务表、默认分类标签、默认文章和测试账号。

### 2.4 环境变量

在项目根目录创建 `.env.local`：

```dotenv
MYSQL_USER=root
MYSQL_PASSWORD=你的MySQL密码
DEEPSEEK_API_KEY=你的DeepSeek API Key
DEEPSEEK_MODEL=deepseek-chat
```

`.env.local` 不应提交到代码仓库。

### 2.5 启动系统

在项目根目录执行：

```powershell
npm install
npm --prefix demo-site install
npm run web
```

或者：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\start-web.ps1
```

启动成功后访问：

```text
http://127.0.0.1:4000/login.html
```

停止系统：

```powershell
npm run web:stop
```

## 3. 登录说明

### 3.1 登录

1. 打开 `/login.html`。
2. 输入用户名和密码。
3. 点击“登录”。
4. 普通用户自动进入用户工作台。
5. 管理员自动进入管理后台。

### 3.2 注册

1. 在登录页找到注册区域。
2. 输入唯一用户名、邮箱和密码。
3. 点击“注册”。
4. 注册成功后使用新账号登录。

注意：

- 用户名不能重复。
- 邮箱必须包含正确的邮箱格式。
- 新账号会自动获得 8 张默认相册图片。

### 3.3 登录失效

当前登录令牌保存在后端内存中。后端重启后，原令牌会失效，需要重新登录。这是课程演示版本的正常行为。

## 4. 普通用户功能使用说明

### 4.1 用户工作台

登录后进入 `/blog.html`。

工作台包含：

- 当前用户、粉丝数和关注数。
- 用户与文章搜索。
- 已公开文章流。
- 分类、标签和归档。
- 我的文章管理。
- 文章发布或编辑。
- AI 写作辅助。

### 4.2 搜索用户和文章

1. 在“搜索用户与文章”输入框输入关键词。
2. 点击“搜索”。
3. 用户结果按用户名或昵称匹配。
4. 文章结果按公开文章标题匹配。
5. 点击用户名进入资料卡。
6. 点击文章标题进入文章详情。

### 4.3 查看资料卡

资料卡显示：

- 用户名和昵称。
- 权限化邮箱：查看自己的资料卡时显示完整邮箱；查看他人资料卡时显示脱敏邮箱；管理员查看任意资料卡时显示完整邮箱。
- 粉丝数和关注数。
- 是否已关注或已互关。
- 个人相册。
- 已公开文章。

在资料卡中可以关注、取关或发私信。

### 4.4 关注、粉丝与互关

1. 点击用户旁的“关注”按钮。
2. 关注成功后按钮显示“已关注”。
3. 如果对方也关注当前用户，按钮显示“已互关”。
4. 工作台顶部的粉丝数和关注数可以点击。
5. 点击后显示对应用户列表。
6. 关闭列表后，右下角索引中的列表目录同步隐藏。

### 4.5 发布文章

1. 进入工作台“发布或编辑文章”区域。
2. 输入标题、摘要和正文。
3. 选择已有分类，或填写自定义分类。
4. 选择或填写标签。
5. 根据需要上传附件。
6. 选择“保存到本地草稿”后点击保存文章，或选择“发布（AI 初审）”提交服务器。

发布结果：

- 正常内容：AI 初审通过，直接公开。
- 风险内容：进入管理员审核，不立即公开。
- 本地草稿：按当前账号保存在当前浏览器中，刷新或重新进入工作台会自动恢复，不会出现在其他设备或“我的文章”服务器列表中。
- 正式发布成功后，系统自动清除对应的本地草稿副本。

### 4.6 使用文章附件

1. 在编辑区域选择附件。
2. 支持图片、PPT、PDF、Word 等常用文件。
3. 单个附件不能超过页面提示的限制。
4. 发布后附件显示在文章详情末尾。
5. 读者可以预览图片或下载附件。

### 4.7 编辑和删除文章

编辑：

1. 在“我的文章”找到目标文章。
2. 点击“编辑”。
3. 修改内容后重新保存。
4. 非草稿内容会再次经过 AI 初审。

删除：

1. 点击目标文章的“删除”。
2. 确认删除。
3. 系统同步删除该文章的评论、点赞、评论点赞、附件、标签关联和原通知。
4. 系统只向受影响用户、管理员和文章作者发送删除通知。

### 4.8 查看文章与目录

1. 点击文章的“查看全文”。
2. 右侧文章目录会识别以下标题：
   - `# Markdown 标题`
   - HTML `h1`—`h6`
   - “一、”“（一）”“第一章”
   - `1.`、`1.1`
   - 独立加粗标题
3. 没有明确标题但存在多个段落时，系统根据段落摘要生成目录。
4. 点击目录项跳转到对应正文位置。

### 4.9 点赞、评论和回复

文章点赞：

1. 点击文章卡片或详情页的“点赞”。
2. 再次操作可取消点赞。
3. 文章作者收到带点赞者用户名的通知。

评论：

1. 在公开文章下输入评论。
2. 正常评论直接公开。
3. 风险评论进入管理员审核。
4. 评论公开后文章作者收到通知。

回复：

1. 点击已公开评论旁的“回复”。
2. 输入回复内容并提交。
3. 回复公开后被回复用户收到通知。

评论点赞：

1. 点击评论旁的“点赞”。
2. 评论作者收到带点赞者用户名的通知。

### 4.10 私信

1. 从用户资料卡、搜索结果、粉丝列表或关注列表点击“私信”。
2. 输入消息并发送。
3. 非互关用户最多向同一用户发送 3 条。
4. 双方互关后发送条数不限。
5. 私信直接发送，不经过 AI 或管理员审核。
6. 对方会收到私信通知。

### 4.11 相册

1. 从侧边栏进入 `/gallery.html`。
2. 登录后显示当前账号的 8 张默认图片。
3. 页面顶部是上传和修改区域。
4. 每张图片下方显示“修改”和“删除”按钮。

上传新图片：

1. 如果相册已有 8 张，请先删除一张。
2. 输入图片标题和说明。
3. 选择 JPG、PNG、WebP 或 GIF 图片。
4. 确认预览后点击“上传图片”。

修改图片：

1. 点击图片下方“修改”。
2. 修改标题或说明。
3. 如需替换图片，重新选择文件。
4. 点击“保存修改”。

删除图片：

1. 点击“删除”。
2. 确认操作。
3. 删除后相册数量减少，可以上传新图片。

### 4.12 动态与通知

1. 侧边栏“动态”旁有红点时表示存在未读消息。
2. 进入动态页面查看通知。
3. 通知中的用户名可以进入发送者资料卡。
4. 点击某条通知的“查看”后，该条消息变为已读。
5. 点击“全部已读”可清除所有未读状态。

通知包括：

- 文章点赞。
- 公开评论。
- 评论回复。
- 评论点赞。
- 新关注者。
- 关注用户发布新文章。
- 私信。
- 内容驳回。
- 文章删除影响通知。

审核通过不会发送多余的“通过”通知。

## 5. 管理员功能使用说明

管理员登录后进入 `/admin.html`。

### 5.1 仪表盘统计

可以查看：

- 用户数量。
- 文章数量。
- 评论数量。
- 分类和标签数量。
- 总阅读量。
- AI 调用数量。

### 5.2 用户管理

可执行：

- 将普通用户设为管理员。
- 将管理员调整为普通用户。
- 封禁或解封用户。
- 删除没有遗留文章和评论的用户。

限制：

- 不能封禁当前登录管理员。
- 不能删除当前登录管理员。
- 用户仍有文章或评论时，应先处理相关内容。

### 5.3 文章审核

1. 在文章管理区查看待审核文章。
2. 阅读标题、正文和 AI 初审结果。
3. 选择“通过上架”或“驳回”。
4. 通过后文章公开，并通知关注者。
5. 驳回后通知文章作者。

### 5.4 评论审核

1. 在评论管理区查看待审核评论。
2. 查看评论内容和 AI 初审结果。
3. 选择通过、驳回或删除。
4. 通过后评论公开，并通知文章作者或被回复者。
5. 驳回后通知评论者。

### 5.5 分类和标签管理

- 新增分类和标签。
- 修改名称或分类说明。
- 删除未被文章使用的分类或标签。
- 已被使用时系统会阻止删除。

### 5.6 AI 日志

AI 日志显示：

- 功能名称。
- 用户输入或提示词。
- 模型处理摘要。
- 返回结果。
- 创建时间。

管理员可按功能和关键词筛选并分页查看。

### 5.7 管理员相册

管理员也拥有独立的 8 张个人相册，可以管理自己的图片，但不能越权修改其他用户的相册。

## 6. AI 功能使用说明

### 6.1 生成大纲

1. 在文章编辑区填写标题和部分正文。
2. 点击“AI 生成大纲”。
3. 等待 DeepSeek 返回结果。
4. 将需要的大纲复制或整理到正文。

### 6.2 生成摘要

1. 填写正文。
2. 点击“AI 生成摘要”。
3. 系统生成适合首页展示的摘要。
4. 根据需要人工修改。

### 6.3 推荐标签

1. 填写标题和正文。
2. 点击“AI 推荐标签”。
3. 系统返回 3—6 个标签。
4. 用户可以继续增加、删除或修改标签。

### 6.4 推荐分类

1. 点击“AI 推荐分类”。
2. 系统优先从已有分类中选择。
3. 如果已有分类不合适，会推荐新分类名称。

### 6.5 博客问答

1. 在文章详情或独立智能体页输入问题。
2. 点击发送。
3. 系统根据已公开文章回答。
4. 返回可能的依据文章。

### 6.6 AI 使用注意事项

- 必须登录后使用 AI。
- AI 返回内容仅作为辅助，发布前应人工检查。
- API Key 无效、模型名错误、额度不足或网络异常时会显示错误。
- 内容初审只负责分流，最终风险判断由管理员完成。

## 7. 常见问题

### 7.1 页面打不开

检查：

1. MySQL 是否启动。
2. 后端 8080 端口是否启动。
3. 前端 4000 端口是否启动。
4. `.env.local` 中数据库账号密码是否正确。
5. 查看 `tmp/run-logs/` 中的日志。

### 7.2 修改前端后页面没有变化

执行：

```powershell
npm --prefix demo-site run generate
```

然后刷新浏览器。必要时使用 `Ctrl + F5` 强制刷新。

### 7.3 后端重启后提示登录失效

当前令牌存储在内存中，后端重启后需要重新登录。

### 7.4 相册不显示上传按钮

- 确认已经登录。
- 相册满 8 张时上传按钮会禁用，请先删除一张。
- 刷新页面并确认 `/gallery.html` 已加载最新脚本。

### 7.5 默认相册不显示

- 重新登录后刷新相册页面。
- 检查 `/images/post/` 下的默认图片资源。
- 检查数据库 `gallery_photos` 和 `user_gallery_settings`。

### 7.6 AI 调用失败

检查：

- `DEEPSEEK_API_KEY` 是否配置。
- `DEEPSEEK_MODEL` 是否有效。
- 网络是否可以连接 DeepSeek。
- API 账户是否有余额。
- 后端 AI 日志中的错误信息。

### 7.7 分类或标签无法删除

该分类或标签仍被文章使用。请先修改或删除引用它的文章。

### 7.8 用户无法删除

该用户仍有文章或评论。管理员需要先处理关联内容。

### 7.9 私信达到上限

非互关用户最多发送 3 条。双方互相关注后即可继续发送。

### 7.10 文章目录只有“正文内容”

建议在正文中使用 Markdown 标题、中文章节标题或数字标题。当前系统也会在存在多个段落时自动生成段落摘要目录。

执行全功能测试：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\blog-system\tests\run-full-system-test.ps1
```

最近一次结果：

```text
前端主要页面：13 / 13 可访问
后端 Maven 打包：通过
端到端业务检查：50 / 50 通过
临时测试数据残留：0
```
